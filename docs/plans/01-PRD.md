> 버전: 1.2.0-draft

> 최종 수정: 2026-03-26
> 작성자: 옥현 (koh-dev)

## 1. 제품 비전

Claude Flow Monitor는 Claude Code의 **진행사항과 워크플로우를 실시간으로 시각화**하는 독립 VS Code 확장프로그램이다. 기존 cc-team-viewer의 Agent Teams 모니터링을 계승하면서, Claude Code의 일반적인 작업 흐름(태스크, 파일 변경, 에이전트 활동)까지 포괄하는 **통합 모니터링 허브**로 발전시킨다.

**프로젝트 포커스**: 현재 열린 VS Code 워크스페이스에 포커스하여 해당 프로젝트와 관련된 Claude Code 활동만 선별적으로 보여주는 **프로젝트 중심 통합 대시보드**를 지향한다. `~/.claude/projects/` 하위의 해시된 프로젝트 디렉토리를 워크스페이스 경로와 자동 매칭하고, 세션 파일 파싱을 통해 Agent Teams뿐 아니라 일반 Claude Code 세션의 도구 호출, 파일 편집, 커맨드 실행 등 모든 AI 활동을 하나의 실시간 피드로 통합 제공한다.

## 2. 문제 정의

### 현재 문제
- Claude Code Agent Teams 사용 시 여러 에이전트의 작업 상태를 한눈에 파악하기 어려움
- 태스크 간 의존성 관계가 복잡해지면 진행 상황 추적이 비효율적
- 에이전트 간 메시지 흐름을 파악하려면 JSON 파일을 직접 열어봐야 함
- 기존 VS Code 확장이 모노레포에 묶여 있어 독립 배포/업데이트가 번거로움

### 목표 사용자

#### 페르소나 1: 파워 유저 개발자 (Primary)
- **프로필**: Claude Code Agent Teams를 일상적으로 사용하는 시니어 개발자
- **컨텍스트**: 3~5개 에이전트를 동시에 운용하며 복잡한 태스크를 병렬 처리
- **핵심 니즈**: 에이전트 상태를 한눈에 파악, 태스크 의존성 추적, 병목 즉시 감지
- **불편 포인트**: JSON 파일을 직접 열어 상태 확인, 에이전트 간 메시지 흐름 추적 어려움
- **성공 기준**: "대시보드만 보면 전체 상황을 5초 안에 파악할 수 있다"

#### 페르소나 2: 팀 리더 / 매니저 (Secondary)
- **프로필**: 멀티 에이전트 워크플로우를 설계하고 진행 상황을 보고하는 역할
- **컨텍스트**: 여러 팀의 에이전트 그룹을 관리, 완료율/처리량 추이 확인 필요
- **핵심 니즈**: 팀별 통계 요약, 완료율 추이, 이상 징후 알림
- **불편 포인트**: 여러 디렉토리를 번갈아 확인, 수동으로 진행률 계산
- **성공 기준**: "별도 보고서 없이 대시보드가 현재 상태를 설명해준다"

#### 페르소나 3: 탐색 중인 개인 개발자 (Tertiary)
- **프로필**: Agent Teams를 처음 사용하거나 소규모(1~2 에이전트)로 실험 중
- **컨텍스트**: 에이전트의 동작 원리를 이해하고 싶어함
- **핵심 니즈**: 직관적 UI, 러닝커브 최소화, 에이전트 활동 시각화
- **불편 포인트**: 에이전트가 "무엇을 하고 있는지" 불투명
- **성공 기준**: "설치 후 1분 안에 에이전트 상태를 확인할 수 있다"

## 3. 목표 (Goals)

### 핵심 목표
1. **독립 VS Code 확장**: cc-team-viewer/core를 의존성으로 사용하되, 독립적으로 빌드/배포 가능한 프로젝트
2. **향상된 시각화**: Stitch 2.0 기반 UI/UX로 더 직관적이고 아름다운 대시보드 제공
3. **VS Code 테마 반응형**: 다크/라이트/고대비 모드 자동 적응 + 커스텀 색상 팔레트
4. **다국어 지원**: 한국어, 영어, 일본어, 중국어 4개 언어
5. **확장 가능한 아키텍처**: 향후 타임라인 뷰, 성능 메트릭, MCP 연동 등 기능 추가 용이

### 비목표 (Non-Goals)
- TUI (터미널 UI) 개발 — 별도 프로젝트로 유지
- Claude API 직접 호출 — 파일 시스템 감시만 수행
- 에이전트 제어 — 읽기 전용 모니터링에 집중
- 웹 대시보드 — VS Code WebView에 집중

## 4. 기능 명세

### 우선순위 프레임워크 (MoSCoW)

| 우선순위 | 정의 | 기능 |
|----------|------|------|

### Phase 1: 핵심 기능 (MVP)

#### F1. 실시간 팀 대시보드 `[Must Have]`
- **Overview 탭**: 팀 전체 통계, 에이전트 카드 (상태/모델/색상/태스크 수)
- **Tasks 탭**: 태스크 테이블 + 칸반 보드 (토글), 상세 아코디언
- **Messages 탭**: 에이전트 간 메시지 스트림, 필터링 (All/Conversation/System), 스레드 그룹핑
- **Deps 탭**: 태스크 의존성 DAG 그래프, 위상 정렬 레이어 기반

**수용 기준 (Acceptance Criteria):**
- [ ] 4개 탭(Overview/Tasks/Messages/Deps) 간 전환이 100ms 이내에 완료된다
- [ ] 스냅샷 업데이트 시 현재 활성 탭만 재렌더링된다 (Dirty Tab 패턴)
- [ ] 칸반 뷰와 테이블 뷰를 토글 버튼으로 전환할 수 있다
- [ ] 메시지 필터링(All/Conversation/System) 적용 시 즉시 목록이 갱신된다
- [ ] DAG 그래프에서 태스크 간 의존성 화살표가 정확히 표시된다

#### F2. 사이드바 트리뷰 `[Must Have]`
- 팀 → 에이전트 → 태스크 계층 구조
- 아이콘으로 상태 표시 (진행중/완료/대기)
- 클릭 시 대시보드 해당 항목으로 이동

**수용 기준:**
- [ ] 팀 → 에이전트 → 태스크 3단계 트리가 올바르게 렌더링된다
- [ ] 새 팀/에이전트 추가 시 트리가 자동으로 갱신된다
- [ ] 트리 아이템 클릭 시 대시보드의 해당 항목으로 스크롤/포커스가 이동한다

#### F3. 상태 바 `[Must Have]`
- 활성 팀 이름 + 완료율 표시
- 클릭 시 팀 선택 퀵피크

**수용 기준:**
- [ ] 상태 바에 "팀명: N% 완료" 형식으로 표시된다
- [ ] 팀이 없을 때 "No active teams" 메시지를 표시한다
- [ ] 클릭 시 퀵피크에 모든 활성 팀 목록이 나열된다

#### F4. 테마 시스템 `[Must Have]`
- VS Code 테마 변수 자동 상속
- 커스텀 시맨틱 토큰 팔레트 (상태별 색상, 에이전트 색상 등)
- 다크/라이트/고대비 3가지 모드 최적화
- CSS 커스텀 프로퍼티 기반 동적 전환

**수용 기준:**
- [ ] Dark+, Light+, High Contrast, High Contrast Light 4가지 기본 테마에서 깨지지 않는다
- [ ] 시맨틱 토큰(`--cfm-*`)이 모든 모드에서 적절한 대비를 제공한다
- [ ] 테마 전환 시 WebView가 즉시 반영된다 (리로드 불필요)

#### F5. 다국어 (i18n) `[Should Have]`
- 한국어 (기본), 영어, 일본어, 중국어
- Settings + UI 버튼으로 전환
- 시스템 언어 자동 감지

**수용 기준:**
- [ ] 4개 언어에서 미번역 키가 0개이다
- [ ] 언어 전환 시 전체 UI가 즉시 갱신된다 (WebView 리로드 없이)
- [ ] VS Code 설정에서 `ccFlowMonitor.language`로 언어를 고정할 수 있다

#### F6. 실시간 알림 `[Should Have]`
- 태스크 완료 시 VS Code 알림
- 에이전트 합류/이탈 알림
- 알림 설정 (ON/OFF) 가능

**수용 기준:**
- [ ] 태스크 완료 알림이 VS Code 정보 메시지로 표시된다
- [ ] 에이전트 합류/이탈 시 알림이 표시된다
- [ ] `ccFlowMonitor.notifications` 설정으로 알림을 끌 수 있다

#### F7. 워크스페이스 매칭 엔진 `[Must Have]`
- VS Code의 workspaceFolder 절대 경로를 기반으로 `~/.claude/projects/` 하위의 해시된 프로젝트 디렉토리를 자동 매칭
- 매칭된 프로젝트의 데이터만 필터링하여 표시

**수용 기준:**
- [ ] 워크스페이스 열 때 자동으로 프로젝트 매칭이 수행된다
- [ ] 매칭 실패 시 수동 선택 UI를 제공한다
- [ ] 전역(`~/.claude/`) + 로컬(워크스페이스 `.claude/`) 디렉토리를 동시에 감시한다

#### F8. 세션 파일 파싱 `[Must Have]`
- Claude Code의 세션 JSONL 파일(`~/.claude/projects/*/sessions/`)을 파싱
- 도구 호출, 파일 편집, 커맨드 실행, 에러 등을 구조화
- Agent Teams 외의 일반 Claude Code 세션도 모니터링

**수용 기준:**
- [ ] JSONL 파싱 실패 시 graceful skip 처리된다
- [ ] 실시간 감시 + 초기 스캔이 모두 동작한다
- [ ] 세션별 활동 통계가 집계된다

#### F9. 통합 Activity Feed `[Should Have]`
- 파일 편집, 커맨드 실행, 태스크 변경, 에이전트 메시지를 하나의 실시간 피드로 통합
- 유형별 필터 토글 (파일/커맨드/태스크/메시지)

**수용 기준:**
- [ ] 새 활동 발생 시 피드에 실시간으로 추가된다
- [ ] 필터 적용 시 즉시 반영된다
- [ ] 최대 200개 항목을 표시한다 (가상 스크롤)

## 5. 성공 지표

| 지표 | 목표값 | 측정 방법 |
|------|--------|-----------|
| 초기 로딩 시간 | < 500ms | WebView ready → 첫 렌더 |
| 스냅샷 업데이트 지연 | < 200ms | 파일 변경 → UI 반영 |
| 메모리 사용량 | < 50MB | VS Code 프로세스 탐색기 |
| 번들 크기 | < 500KB | vsix 패키지 크기 |
| 다국어 커버리지 | 100% | 미번역 키 0개 |

## 6. 기술 요구사항

### 의존성
| 항목 | 사양 |
|------|------|
| VS Code | ≥ 1.90.0 |
| Node.js | ≥ 20.0.0 |
| @cc-team-viewer/core | ≥ 0.1.0 (npm 패키지 또는 로컬 참조) |
| TypeScript | ≥ 5.6.0 |

### 보안 요구사항
- CSP: `script-src 'nonce-{nonce}'` + `worker-src 'none'`
- DOM 조작: innerHTML 금지, textContent + DOM API
- 사용자 입력: escapeHtml() 필수
- localResourceRoots: 빈 배열 (인라인 전략)

### 접근성 (a11y)
- WCAG 2.1 AA 준수 목표
- 키보드 내비게이션 전면 지원
- ARIA 속성 (role, aria-label, aria-expanded 등)
- 고대비 모드 지원

## 7. 디자인 원칙

1. **VS Code 네이티브 느낌**: 확장이 아니라 VS Code의 일부처럼 느껴져야 함
2. **정보 밀도 vs 가독성**: 핵심 정보를 한눈에, 상세 정보는 확장으로
3. **실시간 but 안정적**: 빈번한 업데이트에도 UI가 깜빡이지 않아야 함
4. **점진적 공개**: Overview에서 시작하여 상세 뷰로 drill-down
